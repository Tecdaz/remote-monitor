# Always-on apps and system ambient mode

**URL**: https://developer.android.com/training/wearables/always-on
**Section**: Get started → Battery / Lifecycle

## What this page is

The official guide to **always-on display (AOD)** on Wear — how to make an app remain visible when the user lowers their wrist, how to react to ambient/active transitions, and the battery cost.

## Sections

- Key Concepts (ambient vs active modes)
- System transitions and default behavior
- Customize behavior for the Ambient state
- Control screen-on duration
- Recommendations for Ambient mode
- Debugging and testing
- Example: Workout app

## When to consult

When building a feature that should remain visible (workout tracker, timer, navigation). Required reading for any ambient-mode work.

## Source quote

> Making an app constantly visible significantly impacts battery life, so consider the power impact when adding this feature.
