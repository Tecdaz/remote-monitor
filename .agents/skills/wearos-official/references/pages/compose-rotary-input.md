# Rotary input with Compose

**URL**: https://developer.android.com/training/wearables/compose/rotary-input
**Section**: Get started → Compose

## What this page is

The official guide for **rotary input** on Wear OS in Compose — input from the rotating side button (RSB) or physical/touch bezel. Required for any essential interaction since users average only seconds with the watch.

## Sections

- Scroll indicator (visual feedback for rotary scrolling)
- Custom actions (RSB / bezel mapped to app-specific behaviors)

## When to consult

When designing interactions on a Wear app. Required best-practice: support rotary for any essential interaction, regardless of whether your target watch has RSB / bezel.

## Source quote

> Rotary input refers to input from pieces of your watch that spin or rotate. On average, users spend only a few seconds interacting with their watch. You can enhance your user experience by using Rotary input to allow your user to quickly accomplish various tasks.
