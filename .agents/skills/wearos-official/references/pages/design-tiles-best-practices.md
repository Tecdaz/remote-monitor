# Best practices for tiles

**URL**: https://developer.android.com/design/ui/wear/guides/surfaces/tiles/bestpractices
**Section**: Design → Surfaces → Tiles → Best practices

## What this page is

The opinionated best-practices list for the **Tile surface**. Ten concrete rules: design on black (OLED + contrast), minimum element count, hide titles to ensure tap targets, single primary use-case, at least one container, instantly understandable actions, visual prioritization, container usage, glanceable charts.

## Sections

- Design on black (OLED + contrast rationale)
- Include only necessary elements
- Hide app titles to ensure minimum tap targets
- Choose a single primary use-case to highlight
- Include (at least) one container
- Make actions instantly understandable
- Visually prioritize actions
- Simplify into fewer containers
- Use containers for functional purposes
- Show glanceable representations of graphs and charts

## When to consult

Review checklist for any tile. The "design on black" rule alone is non-obvious and worth citing when explaining why a tile layout looks the way it does.

## Source quote

> Designing on a black background is crucial for Wear OS for two key reasons: [OLED power efficiency + visual contrast].
