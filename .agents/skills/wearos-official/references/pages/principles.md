# Principles of Wear OS development

**URL**: https://developer.android.com/training/wearables/principles
**Section**: Get started → Principles

## What this page is

The **canonical list of Wear OS design/development principles** — the rules every Wear feature should respect. Critical-task focus, wrist-optimization, privacy, surface choice, offline support, and domain-specific guidance for media/fitness/messaging.

## Sections

- Design for critical tasks
- Optimize for the wrist
- Respect the user's privacy
- Use the appropriate surface for the task
- Add notifications to additional surfaces
- Support offline scenarios
- Provide relevant content
- Help users complete a task from another device
- Improve user experience during an app cold start
- Considerations for media apps
- Considerations for fitness apps
- Considerations for messaging apps

## When to consult

Read once at the start of a project and re-skim before major feature work. Use as the spec when a design choice is being debated — "what does the principle say?"
